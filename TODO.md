# مهام: إنشاء صفحة تسجيل دخول منفصلة للمستخدمين

## المهام
- [ ] إنشاء مجلد لعروض المصادقة الأمامية: resources/views/front/auth/
- [ ] إنشاء عرض تسجيل دخول المستخدم: resources/views/front/auth/login.blade.php
- [ ] إنشاء FrontAuthenticatedSessionController للتعامل مع تسجيل دخول المستخدم
- [ ] إضافة مسارات تسجيل دخول المستخدم في routes/web.php
- [ ] تحديث المتحكم للتعامل مع تسجيل دخول المستخدم وإعادة التوجيه إلى لوحة تحكم المستخدم أو الصفحة الرئيسية
- [ ] اختبار تدفق تسجيل دخول المستخدم
- [ ] التأكد من أن تسجيل دخول الإدارة يبقى منفصلاً

## المعلومات المجمعة
- صفحة تسجيل الدخول الحالية تركز على الإدارة بعنوان "لوحة الإدارة"
- AuthenticatedSessionController يعيد التوجيه بناءً على دور المستخدم
- تخطيط الواجهة الأمامية موجود في resources/views/front/master.blade.php
- مسارات الإدارة محمية بـ 'role:admin' middleware

## الخطة
- إنشاء صفحة تسجيل دخول جديدة للمستخدمين، مصممة للمستخدمين وليس الإداريين
- إضافة مسارات جديدة لتسجيل دخول المستخدم (مثل /user/login)
- استخدام طريقة متحكم جديدة أو توسيع الموجود لتسجيل دخول المستخدم
- الحفاظ على تسجيل دخول الإدارة دون تغيير

## الملفات التابعة
- routes/web.php
- resources/views/front/auth/login.blade.php
- app/Http/Controllers/Auth/FrontAuthenticatedSessionController.php (جديد)

## خطوات المتابعة
- التحقق من وظيفة تسجيل الدخول للمستخدمين
- فحص إعادة التوجيه بعد تسجيل الدخول
- التأكد من عدم وجود تعارض مع تسجيل دخول الإدارة
